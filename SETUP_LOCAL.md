# PropEdge V1.0 — Local Setup Instructions

## This folder: `~/Documents/GitHub/PropEdgeV1.0Local`
All Python scripts run from here. Operational folder.

## Requirements
```bash
pip3 install xgboost lightgbm scikit-learn pandas numpy openpyxl nba_api requests keyring
```
V1.0 requires `xgboost` and `lightgbm` in addition to V16's dependencies.

## First-time setup

### Step 1 — Copy data files into source-files/
```
source-files/nba_gamelogs_2024_25.csv
source-files/nba_gamelogs_2025_26.csv
source-files/PropEdge_-_Match_and_Player_Prop_lines_.xlsx   ← 2025-26 props
source-files/PropEdge_2024_25_Match_and_Player_Prop_lines.xlsx
source-files/h2h_database.csv
```

### Step 2 — Copy V12/V14 pkl files into models/
```
models/V9.2/projection_model.pkl
models/V10/projection_model.pkl
models/V11/projection_model.pkl
models/V12/projection_model.pkl
models/V12/direction_classifier.pkl
models/V12/calibrator.pkl
models/V12/segment_model.pkl
models/V12/quantile_models.pkl
models/V14/projection_model.pkl
models/V14/direction_classifier.pkl
models/V14/calibrator.pkl
```
Note: The V17 meta-model (`models/elite/propedge_v1.pkl`) is generated by Step 4.
The old V16 meta-model (`propedge_v1_legacy.pkl`) acts as fallback until V17 is trained.

### Step 3 — Set up GitHub token
```bash
pip3 install keyring
python3 -c "import keyring; keyring.set_password('propedge', 'github_token', 'YOUR_TOKEN')"
python3 run.py token-check
```

### Step 4 — Build season JSONs and train V1.0 model
```bash
python3 run.py generate
```
This runs (~15-20 min):
1. Builds season_2024_25.json and season_2025_26.json
2. Trains V1.0 XGB+LGB+LR ensemble on 33,577+ graded plays
3. Saves models/elite/propedge_v1.pkl
4. Updates player trust scores

### Step 5 — Install scheduler
```bash
python3 run.py install
```

## Daily use
- B0 grades automatically at 07:00 UK via launchd
- B2 predicts automatically at 11:00 UK via launchd
- Manual: `python3 run.py grade` / `python3 run.py predict`
- Health check: `python3 run.py health`
- Force retrain: `python3 run.py retrain`

## Upgrading from V1.0
If you already have V1.0 running:
1. Copy the entire V1.0-Local folder alongside your V1.0-Local folder
2. Copy source-files/ and models/ from V1.0-Local into V1.0-Local
3. Run `python3 run.py retrain` to train the V1.0 model (~10 min)
4. Run `python3 run.py predict` to verify V17 is active
5. Update your launchd plists to point to the V1.0-Local path
   (`python3 run.py uninstall` on V16, `python3 run.py install` on V17)
